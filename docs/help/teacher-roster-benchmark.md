## What this is

The **Benchmark** tab on Teacher Roster shows your students' mastery
**per FAST benchmark / standard**, not just the overall score. Use it
to spot exactly which skills the class is shaky on.

## Reading the grid

- **Rows = students** (same section you picked on Main).
- **Columns = benchmark codes** (e.g. `MA.5.NSO.1.1`), grouped by
  reporting category.
- Each cell shows the student's **% correct** on items tagged to that
  benchmark in the selected PM window. Color shading:
  - **Red** ≤ 50% — weak
  - **Yellow** 51–74% — developing
  - **Green** ≥ 75% — solid

## Filters at the top

- **PM window**: PM1 / PM2 / PM3.
- **Subject**: ELA / Math / Science.
- **Reporting category**: narrow to a single strand.

## Column footer

The bottom row shows **class average per benchmark**. Sort by it to
push the weakest standards to the left — that's your re-teach list.

## Hover & click

- **Hover** a cell to see the item count behind the percentage (e.g.
  "3 of 5 correct, 2 unattempted").
- **Click** a cell to open an item-level drilldown for that student
  on that benchmark.

## Common uses

- **Plan tomorrow's mini-lessons**: sort by class average, pick the
  bottom 3 standards, build small-group rotations.
- **Identify intervention candidates**: a student red across multiple
  related benchmarks is a Tier 2 conversation.
- **Hand off to Class Composer**: if a benchmark is class-wide weak,
  Class Composer (admin/Core Team) can build cross-section intensive
  groups around it.

## Note

Benchmark coverage requires the FAST item-level import to have run.
If the grid is blank, the PM window hasn't been imported with item
data yet — ask your data team.
