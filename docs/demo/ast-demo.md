# Demo Script — AST (Additional Service Time)

A transparent **time-bank ledger** for staff: teachers earn credit for
pre-approved extra duty (Open House, parent conferences, extended meetings) and
spend it later — with **admin approval on both ends**, so the balance is always
defensible.

## 1. The 30-second pitch

When a teacher works beyond contract hours, that time shouldn't live on a sticky
note. PulseEDU gives every staff member an **AST balance** — a running ledger of
**credits** (extra time worked, once an admin confirms it) and **debits** (time
used, pre-approved up front). Everybody sees the same number, the math is
auditable, and admins control the gates.

## 2. Who to log in as / what to open

- **Staff view:** any teacher → the **AST Hub** at `…/ast` to see their own
  balance and ledger.
- **Admin view:** log in as **Chris Clifford** to **approve** earn requests and
  **confirm** completed events (AST approver permission).

## 3. The live demo — step-by-step

**Scene 1 — A teacher's balance.** Open `…/ast` as a teacher. Point to the
**current balance** and the **ledger** of credits/debits in quarter-hour units.
*"Every entry is dated and labeled — no mystery hours."*

**Scene 2 — Earning time.** A teacher requests credit for an event like
**Open House** or a **Parent Conference**. *"It's pending until an admin
confirms the event actually happened."*

**Scene 3 — Admin confirms → credit posts.** As **Chris Clifford**, **confirm**
the completed event. → The **credit** lands in the teacher's ledger and the
balance goes up. *"The admin is the gatekeeper — credit only posts on
confirmation."*

**Scene 4 — Spending time.** Show a **Use** request. *"Debits are pre-approved
up front, so a teacher can't go negative by surprise — the balance reflects
approved usage immediately."*

**Scene 5 — The audit trail.** Scroll the ledger. *"Credits, debits, dates,
reasons — if there's ever a dispute or a bargaining-unit question, the record
is right here."*

## 4. Talking points & objection handling

- **Sales:** removes the spreadsheet/sticky-note chaos and the trust problem
  around comp time.
- **Training:** **two gates** — admin **approves** an earn request and
  **confirms** the event before credit posts; **use** requests debit on
  pre-approval. Same ledger everyone can see.
- *"Is this payroll?"* No — it's the **time-bank record**; it feeds whatever
  process you already use, with a clean audit trail.

## 5. Quick reference

- Staff: **AST Hub** → `…/ast` (balance + ledger, quarter-hour units)
- Admin: **confirm** completed events → credit posts; **approve** use → debit
- Earn examples: Open House, Parent Conference, Extended Faculty Meeting
- Permission: **AST approver** (admin / Core Team)
