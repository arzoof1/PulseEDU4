# PulseEDU — Developer Changes Pack

**Window:** June 19 – June 25, 2026 (plus the PulseDNA Studio audio meter from
June 16, included by request).
**Scope:** Code and feature changes only. Documentation-only commits (user
guides, district brochure, demo-data cleanup, production migration reports) are
intentionally excluded.
**Audience:** Engineers bringing these systems up in a new/empty environment —
each section calls out the **DB changes** and **setup steps** needed to run it.

> **Schema/migration convention (read first).** This repo applies additive
> schema changes via idempotent `ensure*` boot migrations in
> `artifacts/api-server/src/seed.ts` (direct `ALTER TABLE ... IF NOT EXISTS`),
> **not** interactive `drizzle-kit` prompts. On a normal deploy the API server
> boot runs these automatically. For local dev against a fresh DB you can also
> run `pnpm --filter @workspace/db run push`. The Drizzle schema files in
> `lib/db/src/schema/*.ts` are the source of truth.

## Plain-language summary

This week shipped one brand-new module and three significant upgrades, plus a
recording-quality improvement to the family video studio:

- **Eligibility Hub (new).** Tracks attendance-based participation eligibility
  for athletics, clubs, and activities. Coaches/ADs manage rosters; a daily
  attendance upload recomputes who is OK, in the warning zone, or ineligible;
  parents/coaches/principal get notified. Email is live; texting is stubbed.
- **School Tours / Enrollment Leads (upgrade).** Added "Live Tour Capture" (a
  guide walks the building tapping checkpoints on a phone), an automated
  lead-rescue and family follow-up cadence, deeper analytics, and combined tour
  PDFs.
- **Student Profile (upgrade).** Renamed "Student Lookup" to "Student Profile,"
  made it a read-only whole-child snapshot, added a per-student parent note that
  appears on the weekly HeartBEAT, and surfaced teacher room/extension on the
  schedule view.
- **Parent Pick-Up (upgrade).** The front office can now manually override the
  SIS-sourced pickup authorizations (reason-required, audited, temporary or
  permanent), with a reconciliation view when the SIS disagrees.
- **PulseDNA Studio (quality).** The family video recorder now shows a live mic
  level meter with a clipping/peak warning so messages aren't recorded too hot.

## 1. Eligibility Hub — **NEW MODULE**

Attendance-based participation eligibility for athletics / clubs / activities.
Daily upload **replaces** each student's per-semester totals. Manager-facing
tabs: **Activities & Rosters**, **At-Risk Report**, **Parent Notes**, **Upload**.

### Eligibility logic
- **Counted absences** = `fileTotal − min(approvedParentNotes, cap)` plus
  `floor(tardies / ratio)` when `ratio > 0`.
- **Status:** `ineligible` when counted ≥ `ineligibilityThreshold`; `warning`
  when within `warningWindowDays` of (and below) the threshold; otherwise `ok`.
- Ledger is keyed by the **current semester label** — changing the label starts
  a clean count; old rows remain under the old label.

### DB changes
New tables in `lib/db/src/schema/eligibility.ts`:
- `eligibility_activities` — activity/team/club (category: athletics|club|activity).
- `eligibility_activity_members` — roster membership + `jersey_number`.
- `eligibility_activity_coaches` — coach assignments (staff).
- `eligibility_absences` — per student/semester counted totals (replaced on upload).
- `eligibility_parent_notes` — excusal notes (capped per student/semester).
- `eligibility_uploads` — upload audit (filename, matched/unmatched counts).
- `eligibility_notifications` — dedup/digest ledger for sent notices.

New `school_settings` columns (`lib/db/src/schema/schoolSettings.ts`):
- `eligibility_ineligibility_threshold` (default 10)
- `eligibility_warning_window_days` (default 4)
- `eligibility_tardy_to_absence_ratio` (0 = tardies never roll into absences)
- `eligibility_parent_note_cap` (max excusing notes/student/semester, e.g. 5)
- `eligibility_district_ad_notify` (BCC district AD on warning/ineligible)
- `eligibility_semester_*` — current semester label + start/end window

New `staff` column (`lib/db/src/schema/staff.ts`):
- `is_athletic_director` — owns the Eligibility Hub.

### Roles / gates
- `canManageEligibility()` in `artifacts/api-server/src/lib/coreTeam.ts`
  = `isCoreTeam` **OR** `isAthleticDirector` **OR** `capManageDismissal`.
- Settings PUT is further limited to admin / district / SuperUser / AD.
- `publicStaff()` in `auth.ts` now exposes `isAthleticDirector`.

### API (`artifacts/api-server/src/routes/eligibility.ts`, ~21 routes)
- Settings: `GET /api/eligibility/settings`, `PUT /api/eligibility/settings`.
- Activities: `GET/POST /activities`, `PATCH/DELETE /activities/:id`.
- Coaches: `POST /activities/:id/coaches`, `DELETE /coaches/:id`.
- Members: `POST /activities/:id/members`, `POST /activities/:id/members/bulk`,
  `PATCH/DELETE` members; `GET /activities/:id/roster`.
- Attendance: `POST /eligibility/attendance/upload` (REPLACE per student/semester
  + match report).
- Parent notes: list / create (capped, drops one absence) / delete.
- At-Risk report: JSON + PDF endpoints.
- **Tenancy guard:** `assertActivityInSchool(activityId, schoolId)` protects the
  member/coach/roster `:id` routes (bulk insert + roster read); the bulk member
  + attendance uploads resolve students by `local_sis_id` scoped to the school
  and report unmatched IDs back.

### Notifications (`artifacts/api-server/src/lib/eligibilityNotify.ts`)
- Email via Resend: **weekly digest** (cron), **threshold-crossing** alert, and
  **warning re-notify** once per upload while in the warning zone. One combined
  parent message lists all of a student's activities.
- Audience: parents, coaches, principal (warning); district AD optional toggle.
- **SMS is stubbed** (no provider wired).

### Client
- `artifacts/client/src/components/EligibilityHub.tsx` (4 tabs) +
  `EligibilitySettingsPanel`.
- App wiring in `App.tsx`: `activeSection: "eligibility"`, sidebar nav (School
  Admin group), TileHome tile, settings tile, bounce-back. `SettingsHub` +
  onboarding step updated.
- UX: sticky roster header (own bounded scroll region), **Download sample**
  roster CSV (`SIS ID` + optional `Jersey #`), and a dashed bulk-upload
  drop-zone for discoverability.
- **DEVIATION:** the client talks to the server via `authFetch` directly (no
  OpenAPI/Orval codegen), matching the DataImports/Pickup/Ticketing precedent.

### Setup checklist
1. Boot the API server (runs the `ensure*` eligibility migration) or
   `pnpm --filter @workspace/db run push`.
2. Set the school's eligibility settings (threshold, warning window, tardy ratio,
   note cap, **current semester label + dates**) — Settings → Eligibility, or the
   onboarding step.
3. Flag at least one staff member as **Athletic Director** (or rely on Core
   Team / dismissal-manager roles).
4. Email env: `RESEND_FROM_ADDRESS` set, Resend integration connected, and
   `EMAIL_REMINDERS_ENABLED=true` for the digest cron.
5. Create activities, assign coaches, import rosters (sample CSV), then run the
   first attendance upload.

## 2. School Tours / Enrollment Leads — **MAJOR ENHANCEMENTS**

### Live Tour Capture (Phase 4)
A guide scans a QR on the Tour Roadmap (PDF or on-screen) to open a token-gated,
**offline-first, unauthenticated** live-walk screen (`/tour/walk/:token`). The
guide confirms who is guiding (defaults to lead owner), taps once per checkpoint
(client-timestamped, localStorage-buffered, synced when online), and adds
per-stop notes. Timings feed the lead drawer + outcomes metrics.

**DB:** new `tour_walks` table (`lib/db/src/schema/tourWalks.ts`) — one walk per
lead, opaque base62 `token` (globally unique, linkifier-safe), lifecycle
states, per-stop taps with client timestamps. Every query filters on `school_id`.

### Phase 2 "Never lose a lead" (escalation)
- `tours.ts`: lead lifecycle stamps `follow_up_due_at`, `closed_at`,
  `last_escalated_at`, `last_escalated_reason`; a live **"Still deciding"** stage.
- `school_settings`: `tour_first_contact_hours` (default 24),
  `tour_follow_up_business_days` (3), `tour_archive_days` (3),
  `tour_escalation_enabled` (true).
- Background escalation job emails the owner when SLA clocks lapse (idempotent;
  gated globally on `EMAIL_REMINDERS_ENABLED`).

### Phase 3 "Close the loop with families" (nurture)
- `tours.ts` idempotency stamps: `family_reminder_sent_at`,
  `family_thank_you_sent_at`, `family_deciding_nudge_sent_at`.
- `school_settings`: `tour_family_nurture_enabled` (**default OFF** — opt-in),
  `tour_reminder_lead_hours` (24). Cadence: pre-tour reminder, post-tour
  thank-you + survey, "still deciding" nudge, enrollment welcome.

### Other
- `school_settings.tour_sms_scope` (`all` | `urgent`) — limits texts; email
  always sends.
- `tours.ts`: `always_included` checkpoint flag ("School highlight" on every
  tour regardless of family selection).
- New **Tour Guide** role: `staff.cap_tour_guide` — assigned-lead access without
  the full management surface.
- Analytics: Feedback tab, per-guide metrics, "Top wonderings."
- PDFs: **Print Complete Tour Packet** (merges 4 PDFs into one) and a 1-page
  **Quick Roadmap**. Brag-page copy-button feedback + readability fixes.

### Setup checklist
1. Boot/migrate (adds `tour_walks` + the new `tours`/`school_settings` columns).
2. Decide opt-ins: `tour_escalation_enabled`, `tour_family_nurture_enabled`
   (off by default), `tour_sms_scope`, SLA hours/days.
3. `EMAIL_REMINDERS_ENABLED=true` + Resend configured for escalation + nurture.
4. Public-facing URLs/QRs resolve via `publicAppOrigin` (`PUBLIC_APP_URL` →
   `$REPLIT_DOMAINS` → request host) — **never** `$REPLIT_DEV_DOMAIN` in prod.
5. Grant `cap_tour_guide` to volunteer/front-office guides as needed.

## 3. Student Profile (evolved from Student Lookup) — **ENHANCEMENT**

Renamed the "Student Lookup" nav/tile/heading to **"Student Profile"** (internal
`activeSection` key stays `studentLookup`). It renders the whole-child Student
Profile read-only (every edit affordance disabled), under the Teacher Roster
`PrivacyGate`.

### DB changes (`lib/db/src/schema/students.ts`)
- `heartbeat_note` — single current, parent-facing note that surfaces on the
  weekly Friday HeartBEAT (PDF + email block; skipped when empty).
- `heartbeat_note_updated_by` / `heartbeat_note_updated_at` — audit.

### API + behavior
- Visibility-scoped lookup routes (`getVisibleStudentIds`):
  `/api/student-lookup/search`, `GET`/`PUT /:id/heartbeat-note`,
  `GET /:id/schedule`.
- **Invariant:** the note read must use the visibility-scoped lookup route, not
  the school-only `GET /students/:id`, or out-of-roster notes leak.
- The collapsed **Student schedule** block now shows teacher **room number** and
  **phone extension** (`section_roster ⨝ class_sections ⨝ staff`, school-scoped,
  lazy-loaded, remounts per student).

### Setup checklist
1. Boot/migrate (adds the three `students` columns).
2. No env changes; the note appears automatically on the Friday HeartBEAT when
   non-empty.

## 4. Parent Pick-Up — front-office manual override — **ENHANCEMENT**

Pickup authorizations sync from RosterOne (via ClassLink), but the front office
can now override car-tag/rider/auth details by hand: **reason-required (≥5
chars), audited, and temporary (expiry) or permanent**. Office override **wins**
until manually cleared; bulk-assign skips sync-protected rows; the system loudly
flags when RosterOne disagrees.

### DB changes
- `pickup_authorizations` (new columns): `source` (`sis`|`portal`|`manual`),
  `override_reason`, `override_by`, `override_at`, `expires_at`. A row is
  **sync-protected** when `source = 'manual'` OR `override_reason IS NOT NULL`.
- New `pickup_override_audit` table (`lib/db/src/schema/pickupOverrideAudit.ts`)
  — append-only history (actions: `manual_add`, `relabel`, `restrict`,
  `unrestrict`, `deactivate`, `set_expiry`, `dismissal_mode`, `auto_expire`).

### Behavior / roles
- Role gate `canManagePickup`; curb-monitor access added to staff role
  management; dismissal-edit roles fixed.
- Expiry sweep retires past-due temporary overrides (`auto_expire`); the curb
  lookup ignores expired rows immediately.
- Override reconciliation view; **invariant:** "RosterOne disagrees" flag is
  `!sisMayHaveContact` — client badge and server `sisDisagrees` count must agree.

### Setup checklist
1. Boot/migrate (adds the `pickup_authorizations` override columns +
   `pickup_override_audit`).
2. Assign `canManagePickup` / curb-monitor roles to front-office staff.
3. No new env vars.

## 5. PulseDNA Studio — mic level meter + clipping warning *(June 16)*

> Included by request. The audio meter landed June 16, just before the 7-day
> window; the broader PulseDNA Studio (record/review family video messages,
> teleprompter, help guide) shipped June 14–16.

### What changed
`artifacts/client/src/studio/RecordingStudio.tsx` taps the mic with a Web Audio
`AnalyserNode` during recording:
- Computes RMS from the time-domain buffer each animation frame and scales it to
  a **0–1 level meter** (`level` state) for a live fill bar.
- Detects peaks/clipping and raises a **clipping warning** (`clipping` state) so
  the user knows the input is too hot.
- Metering is best-effort (failures ignored) and the rAF loop + analyser are
  torn down on stop/unmount (`meterRafRef` cleanup).

### DB / setup
- **No schema or env changes** for the audio meter itself.
- Context (already in place for the studio): tables `pulse_dna_profiles`,
  `pulse_dna_generations`, `pulse_dna_videos` (`lib/db/src/schema/pulseDna.ts`)
  and `parent_messages`, `parent_message_recipients`
  (`lib/db/src/schema/parentMessages.ts`); routes in `routes/pulseDna.ts` +
  `routes/parentMessages.ts`; a video-purge cron
  (`cron/pulseDnaVideoPurge.ts`); transcode in `lib/videoTranscode.ts`.
- Recording uses `getUserMedia` (mic/camera), which works inside the Replit
  preview iframe; object storage backs uploaded video.

## Consolidated deploy/setup checklist (new environment)

1. **Migrate the DB.** Boot the API server (runs `seed.ts` `ensure*`
   migrations) or `pnpm --filter @workspace/db run push`. This adds: all
   `eligibility_*` tables, `tour_walks`, `pickup_override_audit`, and the new
   columns on `school_settings`, `staff`, `students`, `pickup_authorizations`,
   `tours`.
2. **Email.** Connect the Resend integration, set `RESEND_FROM_ADDRESS`, and set
   `EMAIL_REMINDERS_ENABLED=true` to enable the Eligibility digest, Tours
   escalation, and Tours family nurture crons.
3. **Public URLs / QR codes.** Ensure `PUBLIC_APP_URL` (or `$REPLIT_DOMAINS` in
   prod) is correct so Tours walk/survey QR links don't point at the dev host.
4. **Roles.** Assign `is_athletic_director` (Eligibility), `cap_tour_guide`
   (Tours), and `canManagePickup`/curb-monitor (Pick-Up) as needed.
5. **Per-school config.** Eligibility thresholds + current semester; Tours
   opt-ins (escalation/nurture/SMS scope); verify a default bell schedule exists
   (unrelated modules depend on it).
6. **Verify.** `pnpm run typecheck` clean; eligibility/tours/pickup routes return
   401 unauthenticated; smoke each tab.
