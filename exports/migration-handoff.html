4 · One‑shot data jobs (auto, run once)

Guarded by app_one_shot_markers; each runs exactly once per environment, then no‑ops forever. No action needed: enterprise/licensing fold (re‑applies plans to existing schools), room‑location seed, demo‑email normalization, SuperUser recovery. To re‑run one, delete its marker row and reboot.

5 · Environment variables

Because this is off Replit: the Replit‑injected vars (REPLIT_DOMAINS, REPLIT_DEV_DOMAIN, REPL_IDENTITY, REPLIT_CONNECTORS_HOSTNAME) will be unset. That makes PUBLIC_APP_URL required — it is the fallback used to build all external links and QR codes (e‑sign, tours, kiosk, student/parent portals). Set it to your live HTTPS domain.

🔒 Security — must NOT be enabled in production: STUDENT_DEMO_LOGIN (=1 enables a password‑less student login bypass) and DEMO_MODE. Leave both unset. Review ALLOW_CROSS_DISTRICT_SUPERUSER before enabling.

The AI_INTEGRATIONS_ANTHROPIC_* vars point at Replit's AI proxy and will not work off‑Replit. Only relevant if you use the AI consistency‑check feature; otherwise ignore.

6 · Dependencies, build & run

No runtime dependency or lockfile changes this window. A plain pnpm install after pulling is sufficient.
pnpm install
pnpm run build                 # typecheck + build all packages

# run the API server (boot applies Section 3 auto-migrations)
pnpm --filter @workspace/api-server run start

# the client is a static Vite build — serve artifacts/client/dist behind your proxy;
# API is mounted at /api (path-based routing, same as today)

Node 24 · pnpm · PostgreSQL. The client and API are routed by path (/api → server, everything else → client bundle), so keep your reverse proxy mapping the same as the current live host.

7 · Deploy order & verification

  1. Back up the live database.
  2. Move code (Section 1 patch) and pnpm install + pnpm run build.
  3. Apply Section 2 SQL/push against the live DB.
  4. Set/confirm env vars (Section 5), especially PUBLIC_APP_URL.
  5. Start the api‑server — confirm boot logs show no migration errors (Section 3/4 run here).
  6. Verify:
    • staff has is_confidential_secretary; school_store_redemptions exists; school_settings has the 5 notify_parent_* cols + school_store_inventory_mode.
    • Admin → Family → Parent Notifications loads and a toggle saves (200).
    • A family/student can open the School Store and a redemption creates a row with correct wallet math.
    • Login works for a Confidential‑Secretary user.
    • STUDENT_DEMO_LOGIN is unset; a test parent email sends.

8 · Rollback

Code rolls back by redeploying the prior build. All schema changes here are purely additive (new tables, new nullable/defaulted columns) and backward‑compatible — the previous build keeps working against the new schema, so there is nothing destructive to undo. You generally do not drop anything on rollback.
